Introduce.

이커머스 SaaS에서 1인 프론트엔드로 일하며, 기획부터 개발, 출시, 운영까지 제품이 만들어지는 전 과정을 직접 책임져왔습니다. 없던 기능을 빠르게 제품으로 만들고, 실제 사용자 흐름 안에서 개선해 출시하는 일을 반복해왔습니다. 저는 Zero to One이 낯선 개발자보다, 여러 번 반복해본 개발자에 가깝습니다.

처음에는 구현 속도를 높이는 것이 가장 중요한 역량이라고 생각했습니다. 하지만 AI를 깊게 활용할수록 구현 자체는 점점 덜 어려워졌고, 더 큰 문제는 반복적인 FE 작업 볼륨과 가설 검증 속도를 늦추는 구조라는 걸 분명히 보게 됐습니다. 실제로 AI를 활용한 큰 작업에서, 제가 모르는 것을 모른 채 잘못된 방향으로 밀어붙인 시행착오도 겪었습니다. 그 경험 이후로는 기능을 만드는 데서 그치지 않고, AI가 잘 작동하는 코드 구조와 작업 방식, 개발 환경을 정비하며 FE가 병목이 되지 않는 방향으로 일해왔습니다. 동시에 속도보다 이해와 판단 기준을 먼저 세우는 것이 더 중요하다는 점도 분명히 배웠습니다.

FE 병목을 줄이려다 보니 자연스럽게 백엔드 데이터 구조까지 다루게 됐습니다. 화면만 만들어서는 제품 개발 속도를 바꿀 수 없다고 생각했기 때문입니다. API 응답, 데이터 구조, 기능이 만들어지는 흐름 전체를 함께 보며 FE를 넘어 제품 개발 전반의 병목을 줄이는 방향으로 일해왔습니다. 저는 주어진 화면을 구현하는 프론트엔드 개발자가 아니라, 제품이 더 빠르게 만들어지고 더 자주 검증될 수 있도록 구조를 바꾸는 개발자입니다.
